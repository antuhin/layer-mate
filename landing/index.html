The problem

Messy layers slow everyone down

Every team has a designer whose files look like an unsorted junk drawer. Sometimes that designer is you — and you know it.

🔍

"Which frame was the checkout again?"

When layers are named Frame 47, Group 12, and Rectangle 3, hand-off becomes a treasure hunt. Developers waste time. Designers get DMs at 5pm.

♿

Accessibility issues caught too late

Low contrast text, small touch targets, and tiny fonts slip through until QA — or worse, until a real user with real needs hits a wall.

Redundant wrappers and ghost layers

Hidden layers that ship. Empty frames that confuse. Single-child wrappers nested 6 levels deep. Files bloat. Exports break. Everyone blames "the file".

How it works

From chaos to clean in 4 steps

No configuration. No learning curve. Just open the plugin and go.

01

Install from Community

Find CleanUp Layers in the Figma Community, hit Install — it's free and works immediately.

02

Scan your file

Open the plugin, hit Scan. Get a health score and a breakdown of unnamed, hidden, and empty layers.

03

Preview & rename

Pick a naming convention, preview the before/after, and apply. Undo if anything looks off.

04

Audit & ship

Run the accessibility audit, fix any contrast or token issues, and hand off with confidence.

Naming conventions

Names that fit your workflow

Whether you're building components, writing code, or structuring a design system — there's a naming mode for how you think.

⚛️
Auto Clean
Atomic · Default mode
Frame 1→item
Group 7→container
Frame 47→section

Depth-based structural names. Keeps files consistent without requiring any knowledge of the content.

💻
Dev Ready
Handoff · HTML semantic
Frame (top, full-width)→nav
Vector shape→icon
Frame (full-width)→section

Uses HTML element names — nav, section, article, button. Developers immediately understand the intent.

📚
Library
Component · Figma slash notation
ButtonPrimary→Button / Primary
CardHero→Card / Hero
InputField→Input / Field

Converts camelCase names into Figma slash notation. Perfect for organizing component libraries and variants.

🧠
Semantic
Smart · Pattern detection
Horizontal top frame→top-bar
Circle + image fill→avatar
Frame + shadow + small→popup

Detects real UI patterns from structure, shadows, fills, and dimensions. The most human-readable output.
